Proposed 4-Week RAG Curriculum Structure

Quick Overview

Goal: Master RAG fundamentals and deploy a production-ready AI application in 4 weeks

Time Commitment: ~1 hour/day (7-9 hours/week for weeks 1-3, 14-16 hours for week 4)

What You'll Build:

  • Week 1: Vector similarity search from scratch
  • Week 2: Complete RAG pipeline with optimized chunking
  • Week 3: Multi-agent system with fine-tuning knowledge
  • Week 4: Custom RAG application deployed to production

Overview

This condensed structure focuses on core RAG concepts in weeks 1-3, with week 4 dedicated to building a custom project.

Target Audience: Developers with JS/TypeScript experience

Format:

  • Pre-recorded lessons (asynchronous learning)
  • 1-2 hour live session per week (concepts + live coding)
  • 1 office hour per week (Q&A + debugging help)
  • Slack support throughout

Time Commitment: ~1 hour per day (5-7 hours/week) + live sessions

  • Mon-Thu: Async content (30-45 min/day)
  • Friday: Live session (1.5-2 hours)
  • Weekend: Assignment work (2-3 hours)

Weekly Flow:

  1. Monday-Thursday: Watch videos, read content (45 min/day) - prep for Friday
  2. Friday: Live session (1.5-2 hours) - hands-on coding together
  3. Weekend: Complete assignment (2-3 hours) - apply what you learned
  4. Following Monday: Office hours (1 hour) - get help, preview next week

Starter Project vs Build from Scratch?

Recommended Approach: Hybrid Strategy

Week 1: Build Core Concepts from Scratch

  • Students implement vector math and similarity search themselves
  • No starter code - just TypeScript and OpenAI API
  • Why? They need to understand the fundamentals deeply
  • Time: Manageable in live session + homework

Weeks 2-3: Use Starter Project as Foundation

  • Provide a Next.js starter with:
    • ✅ Basic project structure
    • ✅ API route boilerplate
    • ✅ OpenAI and Pinecone clients configured
    • ✅ TypeScript types defined
    • ❌ NO chunking logic (they build this)
    • ❌ NO agent system (they build this)
    • ❌ NO fine-tuning code (they build this)
    • ❌ NO complete RAG pipeline (they build this)

Week 4: Students Build Fully Custom System

  • Can fork the starter or start fresh
  • Must demonstrate mastery

What's in the Starter Project?

starter-project/
├── app/
│   ├── api/
│   │   └── (empty - students build routes here)
│   ├── components/
│   │   └── chat-interface.tsx (basic UI provided)
│   └── lib/
│       ├── openai.ts (client setup only)
│       ├── pinecone.ts (client setup only)
│       └── types.ts (basic interfaces)
├── scripts/
│   └── (empty - students create scraping scripts)
├── .env.example
├── package.json (dependencies included)
└── README.md (setup instructions)

Philosophy: Give them the scaffolding, but they build the brain.

Why This Approach Works

Pros:

  • ✅ Week 1 from scratch = deep understanding
  • ✅ Weeks 2-3 starter = focus on AI concepts not boilerplate
  • ✅ Students still write 80% of the important code
  • ✅ Faster iteration = more time for learning
  • ✅ Professional workflow (extending existing codebases)

Avoids:

  • ❌ Getting stuck on Next.js config
  • ❌ Wasting time on TypeScript setup
  • ❌ Debugging ESLint instead of learning RAG
  • ❌ Students giving up due to setup issues

Students Still Build:

  1. All chunking logic
  2. All embedding generation
  3. All retrieval logic
  4. All agent systems
  5. All routing/selection logic
  6. All scraping scripts
  7. All custom features
